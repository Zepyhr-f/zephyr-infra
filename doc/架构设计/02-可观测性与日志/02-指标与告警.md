# 指标与告警

> 目的：用可量化指标描述系统健康度，并通过告警做到“问题早发现、可定位、可恢复”。

---

## 1. 核心指标（建议）
- 流量：QPS、并发、请求分布（按接口）
- 时延：P50/P95/P99
- 错误：4xx/5xx 比例、业务错误码分布
- 资源：CPU、内存、GC、线程数
- 依赖：DB 连接池、Redis 延迟、MQ 积压（如有）

---

## 2. 告警建议（摘要）
- 网关 5xx 激增
- 平台/业务服务 P95 时延超阈值
- DB 连接池耗尽、慢 SQL 激增
- Redis 超时/连接错误
- 关键链路错误码（登录失败、鉴权失败）异常升高

---

## 3. 落地工具（示例）
Prometheus + Grafana + Alertmanager（或公司标准平台）。
