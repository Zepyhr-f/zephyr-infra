本目录用于沉淀“排障与监控”相关约定：traceId 贯穿、日志结构、指标与告警、链路追踪。

## 阅读建议
- 先看《日志规范与 traceId》建立排障最小闭环。
- 再看《指标与告警》了解监控口径与阈值建议。
- 需要定位跨服务链路时，查看《链路追踪》。

## 维护规则
- 本目录负责日志、指标、追踪三类可观测规则，避免各模块自行定义字段和 traceId 口径。
- Header 命名、日志字段、MDC 策略变化时，必须同步更新本目录文档。
- 具体技术选型可演进，但外部暴露的排障口径应尽量稳定。
