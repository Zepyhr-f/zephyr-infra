# 链路追踪

> 目的：定位“慢在哪里、错在哪里、依赖出了什么问题”，让排障从“猜”变为“看数据”。

---

## 1. 追踪范围
- 网关 → 平台服务 → 业务服务
- 服务内部：Controller → Service → Mapper/外部依赖
- 外部依赖：MySQL、Redis、HTTP/RPC（尽量）

---

## 2. 关键约定
- traceId 必须贯穿（见《日志规范与 traceId》）
- span 命名：`HTTP {method} {route}` / `DB {table}` / `Redis {cmd}`
- 采样：生产默认采样，关键链路可提高采样率

---

## 3. 工具建议
OpenTelemetry（OTel）作为采集标准，后端存储可选 Jaeger/Tempo/公司平台。
