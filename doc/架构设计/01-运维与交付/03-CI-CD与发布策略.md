# CI/CD 与发布策略

> 目的：定义构建、测试、发布与回滚的标准流程，降低发布风险。

---

## 1. 构建与制品
- Go 服务构建：使用 `go build` 静态编译为无依赖二进制文件
- 制品形式：Docker 镜像（推荐）/ 二进制包（可选）
- 版本号：语义化版本（建议）或公司既定规则

---

## 2. 流水线阶段（建议）
1. Code Check：格式化/静态检查（可选）
2. Unit Test：单测（可选但推荐）
3. Build：构建产物
4. Security Scan：依赖漏洞扫描（可选但推荐）
5. Deploy：部署到 test/staging
6. Smoke Test：冒烟验证
7. Promote：发布到 prod（灰度/滚动）

---

## 3. 发布策略（建议）
- 滚动发布：保持服务可用
- 灰度发布：按用户/租户/比例分流（如网关支持）
- 回滚：保留上一版本镜像与配置快照
